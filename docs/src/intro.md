# 简介

欢迎来到 AliaProxy 官方文档。

**AliaProxy**（简称 Alia）是一个为现代 AI 应用设计的高性能、可扩展的统一 AI 网关。它不仅仅是一个简单的 API 代理，更是一个集成了**多提供商路由**、**统一日志与监控**、**多模态数据持久化**以及**高级流量治理**功能于一体的综合性基础设施。

## 为什么选择 AliaProxy？

在开发 AI 应用时，您可能会面临以下挑战：

- **多供应商管理的复杂性**: 如何在一个应用中无缝切换和使用来自 OpenAI, Anthropic, Ollama 等不同提供商的模型？
- **成本与性能监控**: 如何精确追踪每个请求的 Token 消耗、延迟和费用，并进行可视化分析？
- **API 差异抹平**: 如何用一套标准化的接口（如 OpenAI API）调用所有模型，而无需为每个提供商编写定制化的代码？
- **多模态数据处理**: 如何自动处理和存储图像、音频等多媒体内容？

**AliaProxy** 的核心使命就是为了解决这些问题，让开发者能够专注于业务逻辑创新，而非基础设施的繁琐细节。

## 核心特性

- **OpenAI 兼容接口**: 使用行业标准的 API 格式与所有后端模型进行交互。
- **动态模型路由**: 通过简单的 `provider/model` 格式或自定义别名，将请求智能路由到任何配置的 AI 提供商。
- **负载均衡策略**: 支持在多个模型或 API Key 之间进行轮询（Round-robin）或随机（Random）负载均衡。
- **统一日志与数据分析**: 所有请求的元数据、Token 消耗和内容都会被记录，并可以通过 Web 仪表盘进行深入分析。
- **多模态持久化**: 自动处理图像和音频生成，将二进制数据保存到本地，并通过 API 提供访问。
- **协议自动适配**: 在后台无缝转换不同提供商（如 Anthropic）的专有 API 格式。
- **可视化仪表盘**: 提供一个现代化的 Web UI，用于监控、日志查询和在线模型测试（Playground）。

## 项目愿景

我们致力于将 **AliaProxy** 打造成任何与 AI 模型交互的应用程序的**中央神经系统**。它应该像一个可靠的、智能的路由器，以高效、可观测和可扩展的方式管理所有 AI 流量。

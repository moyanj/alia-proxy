# 用量分析

“用量分析”页面为您提供了基于时间维度的深度数据洞察。您可以选择查看过去 7 天、14 天、30 天或 90 天的数据。

## 关键指标

### 基础统计
- **总请求数**: 选定时间段内的所有 API 调用次数。
- **活跃提供商**: 当前正在处理流量的上游服务数量。
- **使用模型数**: 被调用过的模型种类数量。

### 趋势图表

1.  **API 请求趋势**:
    - **柱状图**: 显示每天的请求总量。
    - **折线图**: 显示每天的请求成功率（Success Rate）。这有助于快速发现某天的服务异常或上游故障。

2.  **错误分布**:
    - 堆叠柱状图，按 HTTP 状态码（如 400, 401, 429, 500）分类展示每天的错误数量。
    - **429 (Too Many Requests)**: 特别关注此指标，它通常意味着您需要增加 API Key 或优化轮询策略。

3.  **模型消耗排行**:
    - 展示 Token 消耗量最大的前 5 个模型。这有助于识别成本的主要来源。

## 峰值监控 (Peak Analytics)

这是一个针对高并发场景设计的高级功能，用于捕捉被平均值掩盖的流量尖峰。

- **RPM 峰值 (Requests Per Minute)**: 显示每分钟内达到的最大请求速率。
- **TPM 峰值 (Tokens Per Minute)**: 显示每分钟内消耗的最大 Token 数量。

**为什么关注峰值？**
很多 AI 提供商（如 OpenAI）的速率限制是基于 RPM/TPM 的。平均值可能看起来很低（例如平均 10 RPM），但如果某一分钟突然飙升到 1000 RPM，依然会触发限流错误。通过这个图表，您可以合理规划容量，设置预警阈值。

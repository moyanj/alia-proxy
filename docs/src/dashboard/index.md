# 仪表盘概览

仪表盘是系统的"驾驶舱"，提供关键指标的实时快照，帮助您快速了解系统运行状态。

## 系统状态卡片

页面顶部展示核心系统状态：

- **已连接提供商**：当前配置的 AI 服务商数量（如 OpenAI、Anthropic、Ollama）
- **总请求数**：系统启动以来的累计请求总量
- **今日请求数**：当天新增请求数
- **活跃模型数**：被调用的不同模型数量 / 系统配置的总模型数

## 实时指标网格

### 成功率
显示 API 请求的成功率百分比，附带：
- **趋势指示器**：显示相比前一期是上升还是下降
- **迷你趋势图**：展示成功率的历史变化曲线

### 平均延迟
显示所有请求的平均响应时间（秒），附带：
- **趋势指示器**：显示延迟变化（数值越小越好，因此趋势颜色与成功率相反）
- **迷你趋势图**：展示延迟的历史变化

### 当前 RPM (Requests Per Minute)
显示每分钟当前的请求速率，附带：
- **速率百分比**：相对于上限的使用率
- **圆形进度指示器**：直观显示负载水平

### 当前 TPM (Tokens Per Minute)
显示每分钟当前的 Token 消耗速率，附带：
- **Token 格式化**：K（千）、M（百万）单位显示
- **圆形进度指示器**：直观显示 Token 消耗水平

## 提供商状态

显示所有配置的 AI 提供商的实时状态：

| 字段 | 描述 |
|------|------|
| 状态图标 | 🟢 绿色=健康 / 🔴 红色=异常 |
| 提供商名称 | 如 `openai-main`、`anthropic-backup` |
| 提供商类型 | 如 `openai`、`anthropic`、`ollama` |
| 请求次数 | 该提供商处理的累计请求量 |
| 成功率 | 该提供商的请求成功率 |
| 平均延迟 | 该提供商的平均响应时间 |

顶部显示健康状态汇总：`X/Y 正常` + 进度条

## Token 消耗排行

展示 Token 消耗量最大的模型 Top 5：

- **排名徽章**：前三名使用金色渐变样式
- **模型名称**：完整模型标识（如 `openai/gpt-4o`）
- **Token 数量**：格式化显示（K/M 单位）
- **进度条**：相对于第一名的占比可视化

## 模型负载分布

支持两种视图模式切换：

### 网格视图 (Grid)
每个模型显示为一个卡片：
- **提供商标签**：彩色胶囊标签（OpenAI=蓝色、Anthropic=紫色、Ollama=橙色）
- **模型名称**：显示模型简称
- **请求数**：累计调用次数
- **迷你趋势柱**：模拟的负载分布图

### 列表视图 (List)
表格形式展示所有模型：
- 按请求数量降序排列
- 显示每个模型的请求数量和占比百分比
- 附带负载趋势可视化柱状图

## 快速分析预览

### 24 小时请求趋势
折线图展示最近 24 小时每小时请求数量变化：
- **X 轴**：0:00 - 23:00
- **Y 轴**：请求数量
- **曲线样式**：平滑蓝色曲线 + 渐变填充区域

### 提供商分布
环形饼图展示各提供商的请求占比：
- **交互式图例**：点击图例可显示/隐藏对应数据
- **悬停提示**：显示具体数量和百分比

## 页面操作

- **刷新状态**：点击刷新按钮重新获取所有统计数据
- **查看日志**：快捷跳转到日志页面
- **查看详情 →**：点击 Analytics 卡片的链接可跳转到完整分析页面
